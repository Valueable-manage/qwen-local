# Qwen 本地部署 GPU 配置指南

按以下步骤操作，可将模型从 CPU 切换到 GPU 运行，显著提升推理速度。

---

## 前置条件

1. **NVIDIA 显卡**（支持 CUDA）
2. **已安装 NVIDIA 驱动**（建议最新版）
3. **已安装 uv**（`pip install uv` 或从 https://github.com/astral-sh/uv 安装）

---

## 第一步：确认显卡和驱动

打开 PowerShell 或 CMD，执行：

```powershell
nvidia-smi
```

- 若能看到显卡型号和驱动版本，说明驱动正常
- 若提示“不是内部或外部命令”，需先安装 [NVIDIA 驱动](https://www.nvidia.cn/Download/index.aspx)

记下右上角 **CUDA Version**（如 12.4），后面安装 PyTorch 时需匹配。

---

## 第二步：关闭所有使用本项目的程序

**必须执行**，否则无法删除虚拟环境：

1. 关闭所有运行 `server.py` 或 `main.py` 的终端
2. 关闭 Cursor 中正在调试的进程
3. 确保没有其他程序在使用 `F:\projects\qwen-local\.venv`

---

## 第三步：删除旧虚拟环境

在 **资源管理器** 中：

1. 打开 `F:\projects\qwen-local`
2. 删除 `.venv` 文件夹

或在 **新开一个 PowerShell** 中执行：

```powershell
cd F:\projects\qwen-local
Remove-Item -Recurse -Force .venv
```

若提示“文件正在使用”，回到第二步，确认所有相关程序已关闭。

---

## 第四步：安装 Python 3.12

PyTorch 的 CUDA 官方包目前不支持 Python 3.13，需使用 3.12。

```powershell
uv python install 3.12
```

等待安装完成。

---

## 第五步：创建虚拟环境并安装依赖

```powershell
cd F:\projects\qwen-local
uv sync
```

该命令会：

- 使用 Python 3.12 创建 `.venv`
- 从 PyTorch CUDA 12.4 源安装 `torch`（约 2.4GB，需一定时间）
- 安装其他依赖

若下载较慢，可多试几次或检查网络。

---

## 第六步：验证 CUDA 是否可用

```powershell
uv run python -c "import torch; print('CUDA 可用:', torch.cuda.is_available()); print('显卡:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else '无')"
```

期望输出类似：

```
CUDA 可用: True
显卡: NVIDIA GeForce RTX xxxx
```

若为 `CUDA 可用: False`，请检查：

- 是否按步骤 1–5 完整执行
- 显卡驱动是否支持当前 CUDA 版本
- 是否使用 Python 3.12（不是 3.13）

---

## 第七步：启动服务

```powershell
uv run python server.py
```

启动时应看到：

```
使用设备: GPU (CUDA)
```

若看到 `使用设备: CPU`，说明仍在使用 CPU，需回到第六步排查。

---

## 常见问题

### Q: 我的 CUDA 不是 12.4 怎么办？

编辑 `pyproject.toml`，将：

```toml
url = "https://download.pytorch.org/whl/cu124"
```

改为对应版本，例如：

- CUDA 12.1：`cu121`
- CUDA 11.8：`cu118`

### Q: 安装时提示找不到 torch 的 wheel？

可能是 Python 版本或平台不匹配。确认：

- 使用 Python 3.12
- 在 Windows 上执行

### Q: 显卡显存不足？

若显存小于 4GB，可尝试：

- 使用更小的模型（如 Qwen3-1.5B）
- 或继续使用 CPU（已自动使用 float32，避免乱码）
