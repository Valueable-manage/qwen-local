# 打包与验证

## 一、打包给客户的内容

### 方式一：运行 pack.ps1（推荐）

```powershell
.\pack.ps1
```

生成 `qwen-local-YYYYMMDD.zip`（约 10–50MB），放在当前目录。

### 方式二：手动打包

将以下内容**打包成 zip**，并**排除**以下目录：

| 必须包含 | 必须排除 |
|----------|----------|
| deploy.bat | .venv |
| deploy_cn.bat | frontend/node_modules |
| setup.ps1 | frontend/.next |
| run.bat | .git |
| create_desktop_shortcut.ps1 | qwen/（模型） |
| start.py, server.py, model_loader.py | __pycache__ |
| main.py, download_model.py | |
| pyproject.toml, uv.lock | |
| icon.ico | |
| frontend/（除 node_modules、.next） | |
| README.md | |

**打包后体积**：约 10–50MB。模型会在客户首次运行 `deploy.bat` 时自动下载（约 8–10GB）。

### 客户使用步骤

1. 解压 zip 到任意目录
2. **无 Python/Node**：先双击 `deploy_prepare.bat` 手动安装，再运行 `deploy.bat`
3. **有 Python/Node**：直接双击 `deploy.bat`（国内网络用 `deploy_cn.bat`）
4. 等待完成（首次会下载模型，约 8–10GB）
5. 完成后会自动启动，并在桌面生成快捷方式

### 网络说明

- **不需要 VPN**，但国内网络可能较慢
- winget 安装 Python/Node 在国内可能卡住 10+ 分钟，建议用 `deploy_prepare.bat` 手动安装
- 模型下载：用 `deploy_cn.bat` 走 ModelScope，国内更快

## 二、如何验证

### 方法 A：本地模拟（推荐）

在**另一台电脑**或**新建用户**下测试：

1. 复制整个项目到测试目录（如 `D:\test-qwen`）
2. 删除以下目录（模拟新电脑）：
   - `.venv`
   - `frontend/node_modules`
   - `frontend/.next`
   - `qwen`（若存在）
3. 双击 `deploy.bat`
4. 观察是否正常完成并启动

### 方法 B：用打包后的 zip 验证

1. 运行 `.\pack.ps1` 生成 zip
2. 将 zip 复制到**另一台电脑**或**新建文件夹**
3. 解压后双击 `deploy.bat`
4. 观察是否正常完成并启动

### 方法 C：虚拟机（最接近真实环境）

1. 新建 Windows 10/11 虚拟机
2. 不安装 Python、Node（或安装后删除）
3. 复制项目 zip 到虚拟机
4. 解压后双击 `deploy.bat`
5. 验证自动安装 Python/Node（如有）及部署流程

### 验证检查清单

- [ ] 双击 deploy.bat 能正常启动
- [ ] 无 Python 时能通过 winget 安装（或提示手动安装）
- [ ] 无 Node 时能通过 winget 安装（或提示手动安装）
- [ ] 模型能自动下载完成
- [ ] 桌面出现 Qwen 快捷方式
- [ ] 程序能正常启动并打开浏览器
- [ ] 能正常对话问答
