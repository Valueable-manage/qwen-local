# CopilotKit 接入详细指南

从当前 `static/index.html`（纯 HTML + 原生 JS）迁移到 CopilotKit 的完整步骤。

---

## 一、架构变化概览

| 当前 | 使用 CopilotKit 后 |
|------|-------------------|
| 单文件 `static/index.html` | React 项目 |
| 原生 JS 直接调 `/api/chat` | 前端 → CopilotKit Runtime → 你的 FastAPI |
| 自定义接口 | 需增加 OpenAI 兼容接口 `/v1/chat/completions` |

**重要**：CopilotKit 是 React 库，必须用 React 写前端，不能继续用纯 HTML。

### CopilotKit 三层架构

```
┌─────────────────┐     runtimeUrl      ┌──────────────────────┐     baseURL      ┌─────────────────┐
│  React 前端     │ ──────────────────► │  CopilotKit Runtime  │ ───────────────► │  你的 FastAPI    │
│  (CopilotChat)  │                     │  (Node.js, 用         │                  │  /v1/chat/      │
│                 │                     │   OpenAIAdapter)      │                  │  completions     │
└─────────────────┘                     └──────────────────────┘                  └─────────────────┘
```

- **Runtime**：负责调用 LLM，使用 `OpenAIAdapter` 的 `baseURL` 指向你的 FastAPI
- **前端**：通过 `runtimeUrl` 连接 Runtime，不直接调 LLM

---

## 二、两种接入方式

### 方式 A：用 CopilotKit CLI 创建（推荐）

一条命令生成 Next.js + Runtime，只需改配置即可对接你的 Qwen。

### 方式 B：手动搭建（Vite + Express）

完全自己搭 React 前端和 Node.js Runtime，更灵活但步骤多。

---

## 三、方式 A：CopilotKit CLI 快速接入（推荐）

### 步骤 A1：创建 CopilotKit 项目

```powershell
cd F:\projects\qwen-local
npx copilotkit@latest create
```

按提示选择：
- Framework: **Next.js**（带内置 Runtime）
- 项目名：如 `qwen-copilot`

会生成 `qwen-copilot/` 目录，内含 Next.js 应用和 CopilotKit Runtime。

### 步骤 A2：在 FastAPI 中增加 OpenAI 兼容接口

在 `server.py` 中新增（在 `if __name__` 之前）：

```python
from pydantic import BaseModel as _BaseModel

class OpenAICompletionRequest(_BaseModel):
    model: str = "qwen"
    messages: list[dict]
    stream: bool = False
    max_tokens: int = 256

@app.post("/v1/chat/completions")
def openai_chat_completions(req: OpenAICompletionRequest):
    """OpenAI 兼容接口，供 CopilotKit Runtime 调用"""
    messages = [{"role": m["role"], "content": m["content"]} for m in req.messages]
    content = chat(messages, max_new_tokens=req.max_tokens)
    return {
        "id": "chatcmpl-local",
        "object": "chat.completion",
        "choices": [{
            "index": 0,
            "message": {"role": "assistant", "content": content},
            "finish_reason": "stop"
        }],
        "usage": {"prompt_tokens": 0, "completion_tokens": 0, "total_tokens": 0}
    }
```

### 步骤 A3：配置 Runtime 使用你的 FastAPI

在生成的 `qwen-copilot` 项目中，找到 Runtime 配置（通常在 `app/api/copilotkit/route.ts` 或类似路径）。修改为：

```ts
import OpenAI from "openai";
import { CopilotRuntime, OpenAIAdapter, copilotRuntimeNextJSAppRouterEndpoint } from "@copilotkit/runtime";

const openai = new OpenAI({
  apiKey: "sk-placeholder",  // 本地模型可随便填
  baseURL: "http://127.0.0.1:8000/v1",  // 指向你的 FastAPI
});

const serviceAdapter = new OpenAIAdapter({ openai, model: "qwen" });
const runtime = new CopilotRuntime();

export const POST = async (req: Request) => {
  const { handleRequest } = copilotRuntimeNextJSAppRouterEndpoint({
    runtime,
    serviceAdapter,
    endpoint: "/api/copilotkit",
  });
  return handleRequest(req);
};
```

### 步骤 A4：配置代理（开发时 Next.js 和 FastAPI 不同端口）

Next.js 默认 3000，FastAPI 8000。Runtime 在服务端运行，直接写 `http://127.0.0.1:8000/v1` 即可，无需代理。

### 步骤 A5：运行

**终端 1**：启动 FastAPI（Qwen 模型）

```powershell
cd F:\projects\qwen-local
uv run python server.py
```

**终端 2**：启动 CopilotKit 前端

```powershell
cd F:\projects\qwen-local\qwen-copilot
npm run dev
```

浏览器访问 `http://localhost:3000`，即可使用 CopilotKit 界面，对话走本地 Qwen。

## 四、方式 B：手动搭建（Vite + Express）

### 步骤 B1：创建 React 项目

```powershell
cd F:\projects\qwen-local
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install
```

### 步骤 B2：安装 CopilotKit

```powershell
cd frontend
npm install @copilotkit/react-core @copilotkit/react-ui @copilotkit/runtime openai
```

### 步骤 B3：创建 CopilotKit Runtime（Express）

在项目根目录新建 `runtime/` 目录：

```powershell
mkdir F:\projects\qwen-local\runtime
cd F:\projects\qwen-local\runtime
npm init -y
npm install express @copilotkit/runtime openai cors
```

创建 `runtime/index.js`：

```js
const express = require("express");
const { CopilotRuntime, OpenAIAdapter, copilotRuntimeNodeHttpEndpoint } = require("@copilotkit/runtime");
const OpenAI = require("openai").default;
const cors = require("cors");

const app = express();
app.use(cors());
app.use(express.json());

const openai = new OpenAI({
  apiKey: "sk-placeholder",
  baseURL: "http://127.0.0.1:8000/v1",
});

const serviceAdapter = new OpenAIAdapter({ openai, model: "qwen" });
const runtime = new CopilotRuntime();

const { handleRequest } = copilotRuntimeNodeHttpEndpoint({
  runtime,
  serviceAdapter,
  endpoint: "/api/copilotkit",
});

app.post("/api/copilotkit", (req, res) => {
  handleRequest(req, res);
});

app.listen(4000, () => console.log("CopilotKit Runtime on http://localhost:4000"));
```

### 步骤 B4：FastAPI 增加 OpenAI 接口

同方式 A 的步骤 A2。

### 步骤 B5：配置 React 前端

修改 `frontend/src/App.tsx`：

```tsx
import { CopilotKit } from "@copilotkit/react-core";
import { CopilotChat } from "@copilotkit/react-ui";
import "@copilotkit/react-ui/styles.css";

export default function App() {
  return (
    <CopilotKit runtimeUrl="http://localhost:4000" agent="default">
      <div style={{ maxWidth: 720, margin: "0 auto", padding: 24 }}>
        <h1>Qwen 问答助手</h1>
        <p style={{ color: "#666" }}>基于 Qwen3.5 + CopilotKit</p>
        <CopilotChat instructions="你是 AI 助手，请用中文回答。" />
      </div>
    </CopilotKit>
  );
}
```

### 步骤 B6：Vite 代理（可选，解决跨域）

若 `runtimeUrl` 用 `http://localhost:4000`，可能跨域。可改为相对路径并代理：

```ts
// frontend/vite.config.ts
export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      "/api/copilotkit": "http://localhost:4000",
    },
  },
});
```

然后 `runtimeUrl="/api"`（或 `/api/copilotkit` 的父路径，视 CopilotKit 文档而定）。

### 步骤 B7：运行

需开三个终端：

1. **FastAPI**：`uv run python server.py`（端口 8000）
2. **Runtime**：`node runtime/index.js`（端口 4000）
3. **前端**：`cd frontend && npm run dev`（端口 5173）

## 五、保留原 static/index.html

- FastAPI 的 `/` 继续返回 `static/index.html`，原界面仍可访问
- CopilotKit 应用单独跑在不同端口（3000 或 5173）
- 或构建后把产物放到 `static/copilot/`，用 `/copilot` 路由提供

## 六、常见问题

### 1. "Failed to fetch" / 连接失败

- 确认 FastAPI、Runtime、前端都在运行
- 确认 `baseURL` 和 `runtimeUrl` 地址正确
- 检查 CORS：FastAPI 若被浏览器直接访问，需加 `CORSMiddleware`

### 2. 返回格式不符合 OpenAI

- 确保 `/v1/chat/completions` 返回 `{ choices: [{ message: { content: "..." } }] }`
- 字段名和层级需与 OpenAI 一致

### 3. 文档上传功能

- 可用 CopilotKit 的 `useCopilotAction` 定义「解析文档」工具
- 或保留 `/api/upload`，在 React 中单独做上传 UI，把解析结果作为上下文传入
